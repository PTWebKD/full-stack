CHƯƠNG 1. GIỚI THIỆU
1.1. Tổng quan dự án
→ Giới thiệu FitFuel+ là nền tảng web kết hợp 3 mảng: Gym Tracking + Food Order + Gear Hub.
→ Điểm cốt lõi: dữ liệu 3 mảng "nói chuyện" với nhau — tập nhóm cơ nào, gợi ý đồ ăn đó.
→ Hai kênh: Customer Portal (Guest & Member) và Back-office Portal (Vendor, Gym Owner, Admin).
→ Điểm kết nối trung tâm: Fitness Passport — hồ sơ thể hình cá nhân ghi lại toàn bộ hành trình.
1.2. Đặt vấn đề
→ Vấn đề 1: Thiếu công cụ theo dõi tiến độ tập luyện — không biết mình có tiến bộ không.
→ Vấn đề 2: Không biết nên ăn gì sau buổi tập — thông tin dinh dưỡng rời rạc, mâu thuẫn, không kết nối với gym log.
→ Vấn đề 3: Thiếu thiết bị tập nhưng không muốn mua mới — không có nền tảng thuê gym gear uy tín tại VN.
→ Vấn đề 4: Thiếu động lực duy trì — tập một mình, không có cơ chế khen thưởng hay cộng đồng.
1.3. Mục tiêu dự án
1.3.1. Mục tiêu kinh doanh
→ Xây dựng hệ sinh thái tạo vòng lặp giá trị: Tập → Gợi ý ăn → Mua/Thuê gear → Tiếp tục tập.
→ Tạo nguồn thu B2B qua Vendor Portal (quán ăn) và Gym Owner Dashboard (phòng tập).
→ Tăng gắn kết người dùng qua Fitness Passport, FitCoin và hệ thống Gamification.
1.3.2. Mục tiêu hệ thống (đo lường được)
→ Log buổi tập và bài tập trong dưới 60 giây.
→ Gợi ý món ăn phù hợp dựa trên nhóm cơ vừa tập (rule-based).
→ Checkout food tối đa 3 bước (cả Guest OTP lẫn Member).
→ Gear Lifecycle hiển thị toàn bộ lịch sử trong 1 màn hình.
→ Trang load dưới 2 giây trên mobile.
1.3.3. Mục tiêu kỹ thuật
→ REST API chuẩn hóa với JWT authentication và phân quyền theo role.
→ Responsive design mobile-first từ 375px, tương thích Chrome, Firefox, Safari, Edge.
→ Continuous Deployment: Frontend → Vercel, Backend → Railway/Render.
→ Bảo mật: bcrypt 10 rounds, HTTPS, chống SQL injection, XSS, CSRF.
1.4. Phạm vi dự án
1.4.1. Trong phạm vi (8 Module)
→ Module 1 — Quản lý tài khoản: Đăng ký, Đăng nhập, Guest OTP, Fitness Passport.
→ Module 2 — Gym Tracking: Session, Log bài tập (Set/Rep/Weight), PR, Smart Suggestion, Streak, Check-in QR.
→ Module 3 — Food Order: Browse, Filter, Cart, Checkout, AI Gợi ý, TDEE, Macro Dashboard, Review có ảnh, Meal Prep.
→ Module 4 — Gear Hub: Listing, Gear Lifecycle, Thuê, Mua, Đăng bán, QR Code, AI Gear Suggestion.
→ Module 5 — Gamification: XP, Level, Badge, Streak, Weekly Challenge, Ranking Board.
→ Module 6 — Social: Feed, Follow, Referral Program.
→ Module 7 — Payment & FitCoin: VNPay/Momo sandbox, Earn/Spend/Deposit FitCoin, gia hạn membership.
→ Module 8 — Admin & B2B: Food Vendor Portal, Gym Owner Dashboard, Admin Panel.
1.4.2. Ngoài phạm vi
→ Wearable device (Apple Watch, Fitbit). Native mobile app (chỉ responsive web).
→ Video call / Live streaming với PT. Chat real-time giữa users.
→ Thanh toán thật (chỉ sandbox). AI/ML phức tạp (chỉ rule-based). Logistics tự động.
1.5. Thông tin thương hiệu
→ Tên: FitFuel+. Slogan: "Train Smart. Eat Right. Gear Up."
→ Logo, màu nhận diện thương hiệu. Chèn hình logo và banner tại đây.
→ Tầm nhìn (Vision): trở thành hệ sinh thái fitness toàn diện tại Việt Nam cho người 18–35 tuổi ở thành phố lớn.
→ Sứ mệnh (Mission): kết nối dữ liệu tập luyện — dinh dưỡng — thiết bị, giúp người dùng đạt mục tiêu thể hình bền vững.
1.6. Phân tích đối thủ cạnh tranh
→ Bảng so sánh FitFuel+ với MyFitnessPal, Strong/JEFIT, Shopee, GrabFood.
→ Tiêu chí so sánh: Gym Tracking, Food Order + Macro, AI Gợi ý từ Gym, Gear Marketplace, Gear Lifecycle, Gamification, FitCoin, B2B Dashboard, Guest Checkout, Fitness Passport.
→ Kết luận: không nền tảng nào kết hợp cả 3 mảng theo cách cho dữ liệu "nói chuyện" với nhau.
1.7. Cấu trúc báo cáo
→ Chương 1 — Giới thiệu: bối cảnh, vấn đề, mục tiêu, phạm vi.
→ Chương 2 — Cơ sở lý thuyết: các công nghệ và framework sử dụng.
→ Chương 3 — Phân tích và thiết kế hệ thống: FR, NFR, Use Case, BPMN, DFD, CSDL, Kiến trúc, Wireframe, Sequence Diagram.
→ Chương 4 — Triển khai website: FE/BE/DB, tích hợp, giao diện, kiểm thử, đánh giá.
→ Chương 5 — Kết luận và hướng phát triển.
CHƯƠNG 2. CƠ SỞ LÝ THUYẾT
2.1. HyperText Markup Language (HTML)
→ Vai trò: nền tảng cấu trúc giao diện Customer Portal và Back-office Portal.
2.2. CSS & Tailwind CSS
→ Utility-first CSS framework, responsive design mobile-first (min 375px), hệ thống design nhất quán, JIT compiler giảm bundle size.
2.3. JavaScript / JSX
→ Xử lý logic phía client, event handling, JSX là cú pháp mở rộng cho React.
2.4. React.js
→ Component-based architecture, Virtual DOM, React Router v6 (Protected Routes), React Hooks (useState, useEffect, useContext).
2.5. Vite (Build Tool)
→ Dev server siêu nhanh với native ES modules. Build production tối ưu cho Vercel deployment.
2.6. Node.js và Express.js
→ Node.js runtime JavaScript phía server. Express.js framework REST API: routing, middleware pipeline, HTTP utilities. Toàn bộ backend FitFuel+ chạy trên stack này.
2.7. MySQL / PostgreSQL và RESTful API
→ Cơ sở dữ liệu quan hệ (ACID compliance, hỗ trợ JOIN phức tạp giữa các module). API chuẩn REST: GET/POST/PUT/PATCH/DELETE, JSON response nhất quán.
2.8. ORM — Sequelize / Prisma
→ Ánh xạ object-relational, quản lý migration, bảo vệ SQL injection, type-safe database access.
2.9. Công cụ phát triển
2.9.1. Visual Studio Code
→ Editor chính, tích hợp ESLint, Prettier, GitLens.
2.9.2. GitHub
→ Version control. Branching: main (production) / dev (tích hợp) / feature/* (từng tính năng). Pull Request workflow.
2.9.3. Postman
→ Kiểm thử và tài liệu hóa API theo từng module (Auth, Gym, Food, Gear, Order, Payment, Admin).
2.9.4. Draw.io / Figma
→ Draw.io: vẽ sơ đồ hệ thống (Use Case, BPMN, DFD, ERD, Sequence Diagram, Architecture). Figma: wireframe giao diện.
2.9.5. Cloudinary
→ Cloud storage ảnh: ảnh đánh giá món ăn (Review) và ảnh thiết bị Gear. SDK tích hợp Node.js.
2.10. Áp dụng lý thuyết vào dự án

→ Khả dụng: Responsive từ 375px, tương thích 4 browser, checkout ≤ 3 bước, font ≥ 14px, contrast ≥ 4.5:1, tap target ≥ 44×44px.
→ Tin cậy: Backup hàng ngày, uptime 99% (demo), lỗi hiển thị thân thiện không lộ stack trace.
→ Quyền riêng tư: Toggle public/private Fitness Passport, toggle từng body photo, quyền xóa tài khoản.

CHƯƠNG 4. TRIỂN KHAI WEBSITE
4.1. Triển khai Frontend
→ Mô tả cấu trúc thư mục React/Vite: /components, /pages, /hooks, /store, /api, /utils, /router.
→ State management: Zustand (cart, auth, notification). Routing: React Router v6 với ProtectedRoute và GuestRoute.
→ API calls: Axios instance với JWT interceptor tự động gắn Authorization header.
4.2. Triển khai Backend
→ Cấu trúc Express.js: /routes, /controllers, /services, /middlewares, /models, /config.
→ JWT: jsonwebtoken, 7 ngày, RS256/HS256. OTP: 6 chữ số, cache TTL 5 phút, khóa sau 3 lần sai 15 phút.
→ bcrypt: 10 salt rounds. Input validation: express-validator. Error handler: global middleware trả JSON chuẩn.
4.3. Triển khai cơ sở dữ liệu
→ MySQL schema với Sequelize migrations. Foreign key ON DELETE CASCADE khi phù hợp.
→ GEAR_LIFECYCLE: append-only, không cho phép UPDATE entry cũ.
→ Seed data demo: 30+ món ăn (đầy đủ macro + tags), 20+ thiết bị gear (kèm 3–5 Lifecycle entries mỗi cái), 50+ bài tập theo 7 nhóm cơ, tài khoản demo (Guest phone, 2 Members, 1 Vendor, 1 Gym Owner, 1 Admin).
4.4. Tích hợp bổ sung
4.4.1. Module AI Gợi ý thực đơn (Rule-based)
→ Hàm recommendFood(session_id): đọc muscle_group → tra mapping object → query FOOD_ITEMS → trả top 3 theo protein density.
→ Fallback recommendByGoal(user_id): đọc user.goal → áp macro filter → query → trả top 3.
→ Không cần thư viện ML. Response time < 80ms (1 DB query với indexed column).
4.4.2. Tích hợp cổng thanh toán VNPay / Momo Sandbox
→ Tạo URL thanh toán với chữ ký HMAC-SHA512 → Redirect sang gateway → Nhận callback → Xác minh chữ ký → Cập nhật trạng thái đơn → Thông báo Vendor.
→ Nhánh FitCoin: trừ số dư trong DB transaction để tránh race condition.
4.4.3. Sinh Gear ID & QR Code
→ Sinh UUID v4 làm gear_id. Dùng thư viện qrcode tạo QR image buffer → upload Cloudinary → lưu secure_url.
→ Frontend dùng html5-qrcode quét QR qua camera trên Vercel HTTPS (camera cần HTTPS permission).
4.4.4. Upload ảnh Cloudinary
→ multer middleware stream multipart/form-data trực tiếp lên Cloudinary (không lưu local disk).
→ Giới hạn: 5MB/file, JPG/PNG/WebP. Review food: 1 ảnh. Gear listing: 2–8 ảnh.
4.5. Giao diện chức năng chính
→ Với mỗi màn hình: chèn ảnh chụp màn hình thực tế + mô tả ngắn chức năng.
4.5.1. Thành phần dùng chung (General Components)
→ Navbar (logo, nav links, cart badge, avatar dropdown). Footer. Toast notification. Loading skeleton. Protected Route wrapper.
4.5.2. Giao diện kênh người dùng (Customer Interface)

CHƯƠNG 5. KẾT LUẬN VÀ HƯỚNG PHÁT TRIỂN
5.1. Kết luận
→ Tóm tắt kết quả đạt được: tỷ lệ hoàn thành FR (~86%), vòng lặp giá trị Gym → AI Food → Gear, Gear Lifecycle minh bạch, Guest OTP checkout, kiến trúc B2B đa vai trò.
→ Xác nhận đạt các NFR: hiệu năng, bảo mật, khả dụng trên môi trường production.
→ Điểm nổi bật: FitFuel+ là nền tảng duy nhất tại VN kết hợp cả 3 mảng theo cách dữ liệu "nói chuyện" với nhau.
5.2. Hạn chế
5.2.1. Cổng thanh toán ở chế độ sandbox
→ VNPay/Momo chỉ sandbox, không xử lý tiền thật. Sandbox đôi khi không ổn định trong demo.
5.2.2. AI rule-based thay vì Machine Learning thực sự
→ Mapping tĩnh không cá nhân hóa theo lịch sử từng user. Chưa có collaborative filtering hay content-based recommendation.
5.2.3. OTP qua console thay vì SMS thật
→ Chưa tích hợp SMS gateway thật (Twilio, ESMS VN) do ràng buộc ngân sách bằng 0.
5.2.4. Ràng buộc free-tier deployment
→ Backend sleep sau 15 phút idle → cold start ~5 giây. Aiven MySQL free tier không có replication.
5.3. Hướng phát triển
5.3.1. Ứng dụng mobile native
→ React Native: tái sử dụng backend API, thêm push notification, QR scan native, offline mode cho gym log.
5.3.2. Tích hợp thiết bị wearable
→ Apple Health / Google Fit API: tự động import workout data, bỏ bước nhập tay.
5.3.3. AI/ML Recommendation thực sự
→ Collaborative filtering (lịch sử tương tác). Content-based filtering (macro profile cá nhân). Hybrid model cho cold-start users.
5.3.4. Production deployment & Containerization
→ Docker + Kubernetes cho backend. Managed DB cluster (AWS RDS). Global CDN cho frontend.
5.3.5. Hoàn thiện hệ thống Gamification
→ Weekly Challenge FE (backend đã xong). Ranking Board real-time. FitCoin economy cân bằng với cơ chế chống lạm phát.
5.3.6. Tính năng real-time
→ WebSocket (Socket.io): thông báo PR real-time, cập nhật trạng thái đơn live. WebRTC: video call với Personal Trainer.
